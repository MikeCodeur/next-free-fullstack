## Comprendre

Nous avons vu que les **React Server Components** s'exécutent côté serveur (backend). Cela signifie que nous pouvons directement interagir avec une base de données depuis nos composants React, sans avoir à passer par une API intermédiaire.

Dans cet exercice, nous allons justement exploiter cette capacité. Pour simplifier la gestion des données, nous utiliserons **LowDB**, une base de données légère qui stocke les informations sous forme de fichier **JSON** sur le serveur. LowDB inclut déjà les fonctions CRUD nécessaires pour manipuler la table `user`, ce qui nous fera gagner du temps.

De base nous avons 2 user : `user@gmail.com` et `admin@gmail.com`

📑 Le liens vers la doc lowDb https://github.com/typicode/lowdb
